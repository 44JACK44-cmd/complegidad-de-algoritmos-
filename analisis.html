Análisis de Complejidad

Constructivo

Tiempo: O(n³)
Espacio: O(n²)
Ventajas:
  • Rápida ejecución
  • Bajo consumo de memoria
  • Genera solución inicial

Búsqueda Local

Tiempo: O(k × n²)
Espacio: O(n²)
Ventajas:
  • Mejora soluciones existentes
  • Control de calidad
  • Escapatoria de óptimos locales

Combinado

Tiempo: O(n³ + k × n²)
Espacio: O(n²)
Ventajas:
  • Mejor calidad de solución
  • Balance tiempo-calidad
  • Robusto para diferentes n

Fuerza Bruta

Tiempo: O((n-1)!)
Espacio: O(n!)
Limitaciones:
  • Solo para n ≤ 4
  • Explosión combinatoria
  • Impráctico para n grandes

Conclusiones y Recomendaciones

Mejor Algoritmo General

El algoritmo Combinado ofrece el mejor balance entre tiempo de ejecución y calidad de solución para la mayoría de los casos.

  • Excelente para n = 4-12
  • Alta tasa de éxito (92%)
  • Tiempo aceptable hasta n=16

Para Resultados Rápidos

El algoritmo Constructivo es ideal cuando se necesita una solución rápida, aunque con menor calidad.

  • Muy rápido (O(n³))
  • Buen punto de partida
  • Adecuado para n grandes

Para Máxima Calidad

Cuando la calidad es prioritaria, usar Búsqueda Local con muchas iteraciones o el algoritmo Combinado.

  • Mejora continua
  • Escapa óptimos locales
  • Control preciso de parámetros

Limitaciones y Consideraciones

El algoritmo de Fuerza Bruta solo es viable para n ≤ 4 debido a la explosión combinatoria.

  • n máximo práctico: 4
  • Útil solo para verificación
  • Base para comparación
